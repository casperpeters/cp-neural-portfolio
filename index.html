FIGURE / RTRBMNEUROPHYSICS
PROJECT / 01 · MSc RESEARCH

Modelling neural dynamics

2020—2022

My MSc research looked at temporal structure in whole-brain neural recordings from zebrafish.

CONTEXT

The recordings contained activity from roughly 100,000 neurons. The task was to identify neural assemblies and describe how their activity changed over time.

WORK

I analysed the data in Python, used dimensionality reduction and ran machine learning experiments on GPU systems.

OUTPUT

The research contributed to a peer-reviewed eLife paper. The associated code and data are public.

PROJECT / 02 · INDEPENDENT PROJECT

Bridgetafel

ACTIVE

An interactive web app for learning bridge through short lessons, practice hands and complete deals.

PROJECT DETAILS +
CONTEXT

Bridge beginners need to learn rules and conventions while also making decisions at the table.

WORK

I designed and built the app, including Dutch NBB Five-card Major bidding logic, practice hands, explanations and the review flow.

CURRENT VERSION

The public version covers bidding, all 13 tricks and a post-game review of the contract, score and key decisions. Unit and Playwright tests cover the main flows.

LIVE DATAPERSONAL PROJECT
PROJECT / 03 · DATA PROJECT

MigratieMonitor

2026

A static dashboard that reads current Dutch migration statistics directly from CBS StatLine.

PROJECT DETAILS +
CONTEXT

The relevant CBS statistics are split across datasets with different time periods and scopes. Monthly national figures and annual migration motives need to remain distinct.

WORK

I selected the datasets and built the client-side OData transforms, charts, responsive interface and GitHub Pages deployment.

CURRENT VERSION

The dashboard shows monthly trends, origin regions and countries, and annual migration motives, with links to the underlying CBS tables.

04 APPROACH

How I tend to work.

Most of my work starts by making an unclear technical problem concrete enough to test.

01 / DEFINE

Define the problem

Write down the terms, assumptions and constraints before choosing a solution.

02 / TEST

Build something testable

A model or small working product usually answers more than another round of abstract discussion.

03 / EXPLAIN

Make it understandable

Document the choices and limitations so someone else can inspect or continue the work.
